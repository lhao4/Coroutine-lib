# 07. 简历与面试表达

## 简历项目描述（可直接使用）

实现了一个约 2000 行的 C++17 协程运行时库，基于 `ucontext` 构建有栈协程切换机制，并实现多线程调度器、`epoll` 事件循环、定时器与系统调用 Hook，将阻塞式 socket IO 转换为协程挂起/恢复流程，形成完整的 IO 协程化执行链路。

## 技术亮点（3-5 条）

1. 基于 `ucontext` 实现协程上下文管理与状态机（READY/RUNNING/TERM）。
2. 实现 `Scheduler + IOManager` 统一调度 CPU 任务、IO 事件和定时任务。
3. 使用 `epoll + eventfd` 支持事件驱动与跨线程唤醒。
4. 通过 Hook `read/write/connect/accept/sleep` 实现阻塞调用协程化。
5. 封装 `FdManager` 管理 fd 元信息与超时控制，保持 IO 行为一致性。

## 面试介绍模板（1-2 分钟）

建议按“问题 -> 设计 -> 关键实现 -> 结果”组织：

- 问题：高并发场景中阻塞 IO + 线程模型的切换与内存成本高。
- 设计：采用有栈协程 + 调度器 + `epoll`，让 IO 等待阶段挂起协程而非阻塞线程。
- 关键实现：`swapcontext` 协程切换、`do_io` 统一处理 `EAGAIN/超时/重试`、事件触发后重入调度。
- 结果：在小规模代码中完整打通协程运行时链路，具备可扩展到生产特性的工程基础。

## 面试追问准备点

1. 为什么选 `ucontext`，以及与 C++20 协程的差异。  
2. 调度公平性、饥饿问题和可选改进策略。  
3. Hook 对第三方库兼容性的边界。  
4. 栈内存占用与性能指标如何量化。  
5. 如果要落地生产，优先改造哪些模块。
