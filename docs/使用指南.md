# 使用指南

## 0. 文档用途
说明当前版本的编译环境、构建命令、最小运行示例和 API 调用方式，覆盖共享栈、协程嵌套、协程池、调度策略。

## 1. 编译环境要求
- 操作系统：Linux（`ucontext`/`epoll` 依赖）
- 编译器：GCC 11+ 或 Clang 14+
- CMake：3.20+
- 构建工具：Ninja 或 Make

## 2. 编译命令

### 2.1 Debug
```bash
cmake --preset debug
cmake --build --preset debug -j
ctest --preset debug --output-on-failure
```

### 2.2 Release
```bash
cmake --preset release
cmake --build --preset release -j
```

## 3. 最小示例运行

### 3.1 共享栈
```cpp
#include <mycoroutine/fiber.h>
#include <mycoroutine/scheduler.h>

int main() {
    mycoroutine::Fiber::SetSharedStackSlotCount(2);

    mycoroutine::Scheduler sc(1, true, "shared_demo");
    sc.scheduleShared([] {
        char guard[256];
        guard[0] = 1;
        mycoroutine::Fiber::GetThis()->yield();
        (void)guard[0];
    }, 64 * 1024);

    sc.stop();
    return 0;
}
```

### 3.2 协程嵌套
```cpp
#include <memory>
#include <mycoroutine/fiber.h>

int main() {
    using mycoroutine::Fiber;
    Fiber::GetThis();

    auto child = std::make_shared<Fiber>([] {
        Fiber::GetThis()->yield();
    }, 0, false, false);

    if (child->call() != Fiber::CALL_OK) return 1;  // 第一次进入，子协程 yield 回父协程
    if (child->call() != Fiber::CALL_OK) return 1;  // 第二次进入，子协程执行到 TERM
    return 0;
}
```

### 3.3 协程池
```cpp
#include <mycoroutine/coroutine_pool.h>

int main() {
    mycoroutine::CoroutinePool pool(8);

    auto f = pool.acquire([] {
        // task
    }, 0, false, false);

    f->resume();
    pool.release(f, 0, false, false);
    return 0;
}
```

### 3.4 调度策略
```cpp
#include <mycoroutine/scheduler.h>

int main() {
    mycoroutine::Scheduler sc(1, true, "policy_demo");
    sc.setPolicy(mycoroutine::SchedulePolicy::HYBRID);

    mycoroutine::ScheduleOptions a;
    a.priority = 1;

    mycoroutine::ScheduleOptions b;
    b.priority = 10;

    mycoroutine::ScheduleOptions c;
    c.deadline_ms = 50;

    sc.scheduleEx([] {}, a);
    sc.scheduleEx([] {}, b);
    sc.scheduleEx([] {}, c);

    sc.stop();
    return 0;
}
```

## 4. API 调用说明

### 4.1 Fiber API
- `Fiber(cb, stacksize, run_in_scheduler, use_shared_stack)`
- `resume()` / `yield()`
- `call()` / `back()` / `parent()`
  - `call()` 返回错误码：`CALL_OK` 为成功，非 0 为失败。
- `SetSharedStackSlotCount()` / `GetSharedStackSlotCount()`

### 4.2 Scheduler API
- `scheduleLock(fc, thread = -1)`
- `scheduleEx(fc, ScheduleOptions)`
- `scheduleShared(cb, stacksize, thread)`
- `setPolicy()` / `getPolicy()`
- `setMLFQConfig()` / `getMLFQConfig()`

### 4.3 CoroutinePool API
- `acquire(cb, stacksize, run_in_scheduler, use_shared_stack)`
- `release(fiber, stacksize, run_in_scheduler, use_shared_stack)`
- `setMaxCachedPerKey()` / `cachedCount()`
