# 06. 优化方向分析

以下建议从系统编程和运行时实现角度出发。

## 1. 调度策略

现状：全局任务容器 + 锁，线程亲和任务依赖遍历筛选。  
优化：改为每线程本地队列 + work-stealing，减少锁竞争与无效扫描。

## 2. 协程栈管理

现状：每协程 `malloc/free` 固定栈。  
优化：
- 引入栈池复用
- 按业务类型配置栈大小
- 添加 guard page，提升栈溢出安全性

## 3. 定时器时钟源

现状：基于 `system_clock`，会受系统时间调整影响。  
优化：改为 `steady_clock`，提升超时行为稳定性。

## 4. Hook 行为可配置性

现状：默认行为依赖调用方是否显式开启 Hook。  
优化：
- 提供统一配置入口（构造参数或全局配置）
- 启动时打印 Hook 状态，降低使用误差

## 5. IO 管理与 fd 生命周期

现状：fd 状态与上下文回收耦合较紧。  
优化：增强 close 路径状态一致性，减少并发下的边界歧义。

## 6. 可观测性与错误处理

现状：部分路径以 `assert` 和 `stderr` 为主。  
优化：
- 统一错误码/异常策略
- 补充关键上下文（fd、event、fiber id、thread id）
- 接入结构化日志或 tracing

## 7. 测试与性能基线

建议新增：
- 协程生命周期单测
- 并发/竞态压力测试
- Hook IO 回归测试
- 基准测试（吞吐、延迟、上下文切换开销）
- CI 中启用 ASan/TSan/UBSan
