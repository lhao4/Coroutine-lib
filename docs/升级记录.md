# 升级记录

## 0. 文档用途
记录本轮对 `mycoroutine` 的升级实现、接口变化、测试结果与当前实现边界。

## 1. 升级总览
本轮完成以下升级：
1. 共享栈支持（Fiber 层）
2. 协程嵌套支持（`call/back`）
3. 复杂调度策略（FIFO/PRIORITY/MLFQ/EDF/HYBRID）
4. 协程池（CoroutinePool）
5. MLFQ 参数可配置（层数、降级、aging）

## 2. 共享栈升级

### 2.1 主要改动
- `Fiber` 构造函数新增参数 `use_shared_stack`。
- 新增 `Fiber::SetSharedStackSlotCount()` / `Fiber::GetSharedStackSlotCount()`。
- `fiber.cpp` 内部新增线程局部共享栈池与快照恢复逻辑。

### 2.2 关键文件
- `include/mycoroutine/fiber.h`
- `src/fiber.cpp`

### 2.3 验证
- `tests/shared_stack/main.cpp`
- 单槽位复用下多协程切换，栈数据一致。

## 3. 协程嵌套升级

### 3.1 主要改动
- 新增 `Fiber::call()`、`Fiber::back()`、`Fiber::parent()`。
- `yield()` 加入优先返回父协程路径。
- `reset()` 清理父子关系状态。
- `call()` 升级为错误码返回，发布版可生效。

### 3.2 关键文件
- `include/mycoroutine/fiber.h`
- `src/fiber.cpp`

### 3.3 验证
- `tests/nested/main.cpp`
- 覆盖父子调用与两层嵌套返回链路。

## 4. 调度策略升级

### 4.1 主要改动
- 新增 `SchedulePolicy` 五种策略。
- 新增 `ScheduleOptions` 与 `MLFQConfig`。
- 新增 `setPolicy/getPolicy`、`setMLFQConfig/getMLFQConfig`、`scheduleEx`。
- `scheduleLock()` 保持兼容并转发到 `scheduleEx()`。

### 4.2 关键文件
- `include/mycoroutine/scheduler.h`
- `src/scheduler.cpp`

### 4.3 验证
- `tests/policy/main.cpp`
- 覆盖 PRIORITY/EDF/HYBRID/MLFQ 与 MLFQ 配置归一化。

## 5. 协程池升级

### 5.1 主要改动
- 新增 `CoroutinePool` 模块与接口：`acquire/release/cachedCount`。
- 调度器回调任务执行路径接入协程池复用。
- 新增调度器池配置接口：`setCoroutinePoolMaxCachedPerKey`、`getCoroutinePoolCachedCount`。

### 5.2 关键文件
- `include/mycoroutine/coroutine_pool.h`
- `src/coroutine_pool.cpp`
- `include/mycoroutine/scheduler.h`
- `src/scheduler.cpp`

### 5.3 验证
- `tests/pool/main.cpp`
- 覆盖复用命中、容量上限、key 隔离。

## 6. 构建与测试变更
- `CMakeLists.txt` 增加 `src/coroutine_pool.cpp`。
- `tests/CMakeLists.txt` 新增 4 个测试目标：
  - `mycoroutine_shared_stack_test`
  - `mycoroutine_nested_test`
  - `mycoroutine_policy_test`
  - `mycoroutine_pool_test`

## 7. 验证结果
执行：
```bash
cmake --build --preset debug -j
ctest --preset debug --output-on-failure
```
结果：`5/5` 测试通过。

## 8. 当前实现边界
- `call()` 路径中，父子协程同时使用共享栈时返回 `CALL_ERR_SHARED_NESTED_UNSUPPORTED`。
