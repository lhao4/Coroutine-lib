# 测试用例

## 0. 文档用途
记录当前版本测试用例、功能验证步骤，以及四项升级点的验证方法。

## 1. 测试目标
- 验证共享栈协程在频繁切换下的数据一致性。
- 验证协程嵌套 `call/back` 返回链路与错误码语义正确。
- 验证复杂调度策略执行顺序符合预期。
- 验证协程池对象复用、容量限制和 key 隔离。
- 验证协程同步原语（CoMutex / Channel / WaitGroup）在并发场景下的正确性与边界行为。

## 3. 单元测试用例说明

### 3.1 共享栈复用测试
- 执行方式：`mycoroutine_shared_stack_test`
- 方法：
  - `Fiber::SetSharedStackSlotCount(1)` 强制单槽位。
  - 创建多个共享栈 Fiber，循环 `yield/resume`。
  - 在局部数组写入 pattern，恢复后逐字节断言。
- 通过标准：所有 Fiber 结束且断言通过。

### 3.2 协程嵌套测试
- 执行方式：`mycoroutine_nested_test`
- 方法：
  - 用 `trace` 验证 `parent->child->parent` 顺序。
  - 验证两层嵌套 `root->mid->grand` 顺序收敛。
  - 新增失败路径：父子协程同时共享栈时，`call()` 返回 `CALL_ERR_SHARED_NESTED_UNSUPPORTED`。
- 通过标准：trace 匹配预期向量，错误码匹配预期值。

### 3.3 调度策略测试
- 执行方式：`mycoroutine_policy_test`
- 方法：
  - `PRIORITY`：高优先级先执行。
  - `EDF`：更早 deadline 先执行。
  - `HYBRID`：deadline 优先，其次 priority。
  - `MLFQ`：长任务 yield 后降级，短任务插队执行。
  - `MLFQConfig`：`levels=0` 与 `aging_sequence_gap=0` 归一化到 1。
- 通过标准：执行顺序与断言向量一致。

### 3.4 协程池测试
- 执行方式：`mycoroutine_pool_test`
- 方法：
  - 复用命中：释放后再次获取对象地址一致。
  - 上限控制：`max_cached_per_key=1` 时缓存不超过 1。
  - key 隔离：`use_shared_stack=false/true` 缓存桶独立。
- 通过标准：缓存计数、地址复用与状态断言通过。

### 3.5 协程同步原语测试
- 执行方式：`mycoroutine_sync_test`
- 方法：
  - CoMutex + WaitGroup 并发：4 线程 64 协程各加锁 300 次，WaitGroup 等待全部完成。
  - Channel 有缓冲：容量 2，单生产者发送 24 条消息，单消费者按序接收。
  - Channel 无缓冲：同步通道发送 8 条消息，验证严格交替收发。
  - tryLock 边界：持锁时其他协程 tryLock 返回 false。
  - close 后 send：关闭通道后 send 返回 false，缓冲区剩余数据仍可 recv，排空后 recv 返回 false。
  - 多生产者多消费者：4 生产者 x 50 条 + 4 消费者，验证总量一致。
  - WaitGroup 零计数：未调用 add 时 wait 立即返回。
  - 重复 close：连续两次 close 不崩溃。
  - Scheduler 生命周期：waiter 存活时间超过 Scheduler，done() 不触发 UAF。
- 通过标准：所有断言通过，TSan 无数据竞争。

### 3.6 Hook 功能测试
- 执行方式：`mycoroutine_hook_test`
- 方法：
  - 验证 Hook 模块对 `sleep`、`socket`、`connect`、`accept`、`read`/`write` 等系统调用的协程化拦截。
  - 验证 `IOManager` 事件注册与回调恢复路径。
  - 验证 `TimerManager` 定时器触发与取消。
  - 验证 `FdManager` 对 fd 元信息（socket/nonblock/timeout）的管理一致性。
- 通过标准：所有 Hook 场景断言通过，事件回调与定时器行为符合预期。

| 优化点 | 测试方法 | 验收标准 |
|---|---|---|
| 共享栈复用 | 单槽位多协程交替切换 + 栈数组校验 | 无数据污染，全部协程结束 |
| 嵌套协程切换 | 单层/双层 `call()` 链路与错误码校验 | 顺序正确、错误码正确 |
| 调度策略 | PRIORITY/EDF/HYBRID/MLFQ 执行顺序断言 | 顺序与策略定义一致 |

| Hook 协程化 | Hook 拦截、IOManager 事件、定时器、FdManager 断言 | Hook 行为正确、事件回调正常 |
