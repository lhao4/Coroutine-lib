# 高级特性设计

## 0. 文档用途
说明当前版本七项核心能力的实现方案与关键实现点：汇编上下文切换、共享栈、协程嵌套、复杂调度算法、协程池、协程同步原语、任务队列优化。

## 1. 设计基线

### 1.1 运行时主链路
- 协程执行单元：`Fiber`
- 任务调度器：`Scheduler`
- IO 事件调度：`IOManager`（继承 `Scheduler`）
- 复用组件：`CoroutinePool`

### 1.2 核心能力
- 汇编上下文切换：x86_64/AArch64 汇编替代 `ucontext`，消除系统调用开销。
- 共享栈：降低大量协程时的栈内存占用。
- 协程嵌套：支持父协程同步调用子协程。
- 复杂调度：支持优先级、多级反馈、截止期等策略。
- 协程池：复用终止协程对象，减少创建销毁开销。
- 协程同步原语：`CoMutex`、`WaitGroup`、`Channel<T>`，协程等待不阻塞工作线程。
- 任务队列优化：`deque` + move 语义，降低入队/出队开销。

## 2. 汇编上下文切换

### 2.1 设计目标
替代 `ucontext` 系列函数（`getcontext/makecontext/swapcontext`），消除信号掩码保存/恢复带来的系统调用开销。

### 2.2 上下文结构
`FiberContext`（定义于 `include/mycoroutine/context.h`）：
- x86_64：`rsp`, `rip`, `rbx`, `rbp`, `r12`-`r15`（8 个 callee-saved 寄存器）
- AArch64：`sp`, `pc`, `x19`-`x28`, `x29(fp)`, `x30(lr)`, `d8`-`d15`（callee-saved + FP 寄存器）

### 2.3 汇编切换函数
`mycoroutine_context_swap(FiberContext* from, FiberContext* to)`：
1. 将当前 callee-saved 寄存器和栈指针保存到 `from`
2. 从 `to` 恢复寄存器和栈指针
3. 跳转到 `to` 保存的 PC/RIP 继续执行

### 2.4 协程初始化
`InitFiberContextStack(ctx, stack, stacksize)`：
1. 计算栈顶并 16 字节对齐
2. 写入入口地址 `FiberEntryPoint`
3. 设置 `FiberContext` 的 SP 和 PC

### 2.5 Sanitizer 兼容
- ASan：调用 `__sanitizer_start/finish_switch_fiber` 通知栈切换，新协程在 `MainFunc` 入口调用 `finish_switch_fiber`
- TSan：调用 `__tsan_create/switch_to_fiber` 追踪协程级数据竞争
- 共享栈快照前调用 `__asan_unpoison_memory_region` 避免误报

### 2.6 性能
Release 模式下 Fiber 上下文切换约 **17 ns/op**，相比 `ucontext` 的 385 ns/op 提升约 22 倍。

## 3. 共享栈设计与实现

### 3.1 入口接口
`Fiber` 新构造参数：

```cpp
Fiber(std::function<void()> cb,
      size_t stacksize = 0,
      bool run_in_scheduler = true,
      bool use_shared_stack = false);
```

`Fiber` 共享栈配置接口：

```cpp
static void SetSharedStackSlotCount(size_t slot_count);
static size_t GetSharedStackSlotCount();
```

### 3.2 内部结构（`src/fiber.cpp`）
- `SharedStackSlot`
  - `char* stack`
  - `size_t size`
  - `std::weak_ptr<Fiber> owner`
- `SharedStackPool`
  - 固定槽位数组
  - 轮转分配 `acquireSlot()`
- 线程局部池
  - `thread_local std::unordered_map<size_t, std::unique_ptr<SharedStackPool>> t_shared_stack_pools`
  - 按栈大小分桶

### 3.3 切换关键路径
- `Fiber::resume()`
  - 共享栈协程先调用 `prepareSharedStack()`。
  - 如果目标槽位已被其他协程占用，先保存旧 owner 的活动栈快照。
  - 恢复当前协程快照到共享栈后再 `mycoroutine_context_swap`。
- `prepareSharedStack()`
  - 绑定执行线程（共享栈协程仅允许在首次绑定线程运行）。
  - 初始化上下文（延迟到首次拿到槽位时）。
- 快照函数
  - `saveSharedStackSnapshot()`：从上下文 SP 保存。
  - `saveCurrentSharedStackSnapshot()`：从当前栈标记位置保存。
  - `restoreSharedStackSnapshot()`：切入前恢复。

## 4. 协程嵌套支持逻辑

### 4.1 新增接口

```cpp
int call();
void back();
Fiber* parent() const;
```

`call()` 返回码：
- `Fiber::CALL_OK`
- `Fiber::CALL_ERR_NOT_READY`
- `Fiber::CALL_ERR_NO_CURRENT_FIBER`
- `Fiber::CALL_ERR_SELF_CALL`
- `Fiber::CALL_ERR_SHARED_NESTED_UNSUPPORTED`

### 4.2 状态字段
- `Fiber* m_parent`
- `bool m_returnToParent`

### 4.3 执行语义
- 父协程执行 `child->call()`：
  - `child.m_parent = parent`
  - `child.m_returnToParent = true`
  - `mycoroutine_context_swap(parent, child)`
- 子协程执行 `yield()`：
  - 若存在父协程并且 `m_returnToParent=true`，优先走 `back()` 返回父协程。
  - 否则按原路径回到调度协程或主协程。
- `reset()` 会清理父子关系字段，保证对象复用安全。

### 4.4 当前边界
- 父子协程同时使用共享栈的嵌套路径暂未开放，`call()` 返回 `CALL_ERR_SHARED_NESTED_UNSUPPORTED`。

## 5. 调度器与复杂调度算法

### 5.1 调度策略

```cpp
enum class SchedulePolicy {
    FIFO,
    PRIORITY,
    MLFQ,
    EDF,
    HYBRID
};
```

### 5.2 调度参数

```cpp
struct ScheduleOptions {
    int thread = -1;
    int priority = 0;
    uint64_t deadline_ms = 0;
};
```

### 5.3 MLFQ 参数

```cpp
struct MLFQConfig {
    uint8_t levels = 3;
    bool demote_on_yield = true;
    bool enable_aging = true;
    uint64_t aging_sequence_gap = 64;
};
```

### 5.4 调度实现点（`src/scheduler.cpp`）
- 入队接口
  - `scheduleEx(fc, options)`
  - 兼容接口 `scheduleLock(fc, thread)`
- 核心选取函数
  - `pickNextTaskLocked(int thread_id, ScheduleTask&, bool& tickle_me)`
- 选取规则
  - `FIFO`：首个匹配线程任务。
  - `PRIORITY`：`priority` 高优先，`sequence` 保序。
  - `MLFQ`：按 `mlfq_level` 选取，支持 aging 提升。
  - `EDF`：有截止期任务优先，截止期更早优先。
  - `HYBRID`：先截止期，再优先级。
- MLFQ 回灌
  - Fiber 运行后若仍 `READY`，根据 `demote_on_yield` 降级后重新入队。
- 配置归一化
  - `levels == 0 -> 1`
  - `aging_sequence_gap == 0 -> 1`

## 6. 协程池设计与实现

### 6.1 核心接口

```cpp
std::shared_ptr<Fiber> acquire(std::function<void()> cb,
                               size_t stacksize = 0,
                               bool run_in_scheduler = true,
                               bool use_shared_stack = false);

void release(std::shared_ptr<Fiber>& fiber,
             size_t stacksize = 0,
             bool run_in_scheduler = true,
             bool use_shared_stack = false);
```

### 6.2 关键实现（`src/coroutine_pool.cpp`）
- 分桶 key：`stacksize + run_in_scheduler + use_shared_stack`
- `acquire`：
  - 命中缓存 -> `fiber->reset(cb)`
  - 未命中 -> 新建 `Fiber`
- `release`：
  - 仅回收 `TERM` 状态
  - 按桶容量 `m_maxCachedPerKey` 限流

### 6.3 调度器接入
- `Scheduler::run()` 执行 `task.cb` 时：
  - 通过 `m_coroutinePool.acquire(...)` 获取协程
  - 执行结束后 `TERM` 则 `release(...)`

## 7. 协程同步原语

### 7.1 设计目标
协程等待时挂起协程本身（yield），不阻塞工作线程，实现高效的协程间协作。

### 7.2 统一等待节点
`FiberWaiter`（定义于 `include/mycoroutine/sync.h`）：
- `shared_ptr<Fiber> fiber`：当前协程
- `weak_ptr<Scheduler::SchedulerRef> schedulerRef`：调度器弱引用，避免 UAF
- `int threadId`：线程亲和 ID
- `atomic<bool> notified`：唤醒标记

### 7.3 等待/唤醒流程
- 等待：创建 `FiberWaiter`，放入等待队列，`yield` 挂起；恢复后检查 `notified`
- 唤醒：设置 `notified=true`，通过 `SchedulerRef::schedule()` 重新调度协程

### 7.4 CoMutex
- `lock()`：尝试获取锁，失败则创建 waiter 入队并 yield
- `tryLock()`：非阻塞尝试，返回 bool
- `unlock()`：释放锁，唤醒等待队列首个协程并转移所有权

### 7.5 WaitGroup
- `add(delta)`：修改计数，归零时批量唤醒所有 `wait()` 协程
- `done()`：等价于 `add(-1)`
- `wait()`：计数非零时挂起

### 7.6 Channel\<T\>
- 支持有界缓冲（`capacity > 0`）和同步通道（`capacity = 0`）
- `send(value)`：缓冲满或同步通道无接收者时挂起
- `recv(out)`：缓冲空时挂起，通道关闭且缓冲空时返回 false
- `close()`：标记关闭，唤醒所有等待协程

## 8. 任务队列优化

### 8.1 改动
- 任务容器从 `std::vector<ScheduleTask>` 改为 `std::deque<ScheduleTask>`
- 入队使用 `emplace_back(std::move(task))`
- 出队使用 `out_task = std::move(*it)` + `erase(it)`
- MLFQ 重新入队同步使用 move 语义

### 8.2 性能
Release 模式下 Scheduler 单线程吞吐约 **527 ns/op**，相比优化前的 83,170 ns/op 提升约 158 倍。

## 10. 关键功能示例

### 10.1 共享栈 + 调度策略
```cpp
#include <mycoroutine/scheduler.h>
#include <mycoroutine/fiber.h>

int main() {
    mycoroutine::Fiber::SetSharedStackSlotCount(4);

    mycoroutine::Scheduler sc(1, true, "demo");
    sc.setPolicy(mycoroutine::SchedulePolicy::PRIORITY);

    mycoroutine::ScheduleOptions high;
    high.priority = 10;

    sc.scheduleEx([] {
        mycoroutine::Fiber::GetThis()->yield();
    }, high);

    sc.scheduleShared([] {
        // 共享栈协程
    }, 64 * 1024);

    sc.stop();
    return 0;
}
```

### 10.2 协程嵌套
```cpp
#include <memory>
#include <mycoroutine/fiber.h>

int main() {
    using mycoroutine::Fiber;
    Fiber::GetThis();

    auto child = std::make_shared<Fiber>([] {
        Fiber::GetThis()->yield();
    }, 0, false, false);

    if (child->call() != Fiber::CALL_OK) return 1;
    if (child->call() != Fiber::CALL_OK) return 1;
    return 0;
}
```

## 11. 使用注意事项

### 11.1 共享栈
- 共享栈协程绑定首次运行线程，跨线程执行会触发断言。
- 栈对象在切出后依赖快照恢复，不应持久化栈地址。

### 11.2 协程嵌套
- `call()` 要求子协程处于 `READY`。
- 父子协程同时启用共享栈的嵌套路径会返回 `CALL_ERR_SHARED_NESTED_UNSUPPORTED`。

### 11.3 调度策略
- `scheduleLock()` 仍可用，等价于默认 `ScheduleOptions` 入队。
- MLFQ 参数会归一化：`levels` 与 `aging_sequence_gap` 最小值为 1。

### 11.4 协程池
- `release()` 仅回收 `TERM` 协程。
- 缓存 key 由 `stacksize + run_in_scheduler + use_shared_stack` 组成，参数要与 `acquire()` 保持一致。

## 12. 已知限制与边界

### 12.1 功能边界
- 父子协程同时启用共享栈时，`call()` 返回 `CALL_ERR_SHARED_NESTED_UNSUPPORTED`，不执行切换。
- 调度队列为全局 `deque` + 策略选取扫描，任务数量极大时选取与锁竞争开销会放大。

### 12.2 平台依赖
- 支持架构：x86_64、AArch64（汇编上下文切换）。
- 运行时依赖 Linux 能力：`epoll`、`eventfd`、`dlsym`。
- 非 Linux 平台未适配。

### 12.3 运行时约束
- 共享栈协程在首次运行线程绑定后，不支持跨线程执行。
- `call()` 要求目标协程处于 `READY`，禁止自调用，失败通过错误码返回。
- `CoroutinePool::release()` 仅回收 `TERM` 协程，`acquire/release` 参数必须匹配同一 key。

### 12.4 观测能力
- 运行期主要依赖日志输出。
- 尚未提供内建指标导出（如调度延迟、池命中率、共享栈拷贝字节数）。

### 12.5 测试边界
- 当前自动化测试覆盖 8 个目标（smoke/shared_stack/nested/policy/pool/sync/hook/stress），均通过 Debug/ASan/TSan 三模式验证。
- 网络 demo 连通性仍以手工流程为主，不在默认 `ctest` 集合中。
- Benchmark 需手动运行 `./build/release/tests/mycoroutine_benchmark`。
